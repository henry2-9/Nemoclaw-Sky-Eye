## 0. 北極星重校(這版的關鍵)
評審主軸是 **autonomous / no human in the loop**。上一版把「自然語言操作台(人去問它)」當主力是**方向錯誤**——那是 human-in-the-loop。本版一切圍繞:

> **「啟動後由排程持續監看。它自己看、自己想、自己調查、自己治理、自己決策、自己處置、自己報告，且每次觸發來源可稽核。」**

## 1. 戰略判斷
**當時規劃基線:** 真 NemoClaw 治理、防注入測試、flight recorder、live CCTV、單台 GB10 級聯、SQLite、dashboard、supervisor 排程。

**第一名差距(兩點):**
1. **自主性要「更深 + 可稽核」**:目前閉環偏「偵測→通知」一槍;要升級成「偵測→**自主調查**→治理→**自主處置/報告**→**健康監測**」的完整閉環,且 dashboard 要呈現觸發來源。
2. **要直觀**:評審 10 秒要看懂「這是一個自己在跑的 AI 保全官」,不是一張要讀的表。

## 3. 優先序(全以「推進自主性」為準)

### P0-1 ·【閉環】自主分級處置 + 自動事件報告
- **做什麼**:agent 不止通知。確認事件後**自主**執行分級動作(log / notify / **escalate 二級告警** / **critical 自動產 Markdown 事件報告**),全由 NemoClaw 政策決定、無人核准。
- **為何贏**:把 autonomous 從「會看會說」推到「**會做**」——閉環關鍵。
- **工時** ~0.5–1 天 · 低風險(串接既有 violation-report + policy actions)。

### P0-2 ·【可稽核 + 直觀】自主運行監控首頁 + 觸發來源呈現
- **做什麼**:新首頁(B 重新定位為**自主運行觀測窗**,非互動控制台):
  - 首頁呈現:**「自主處置 · 無人工核准流程 · 已連續 Xh · 處理 N 起」**,並統計排程與手動 demo 觸發數。
  - **live 攝影機格**(週期快照,不連續解碼)+ **威脅等級** banner。
  - **agent 思考即時流**:最新事件級聯逐步動畫(① Falcon→② Nemotron→③ NemoClaw→④ 政策→⑤ 處置),讓「它自己在跑」看得見。
- **為何贏**:10 秒看懂 + 用 audit 直接交代事件來源與處置路徑。
- **工時** ~1 天 · 風險:抓幀效能(僅週期快取)· 複用:flight recorder、analyze 抓幀、glass UI。

### P0-3 ·【agentic 深度】自主調查(不確定就自己挖)
- **做什麼**:Nemotron 信心邊界 / 場景含糊時,agent **自主**再調查——多抓幾幀 / 換角度 / 追問 Nemotron——再決策。調查步驟記入 flight recorder。
- **為何贏**:這才是「agent」——自我導向多步推理,不是固定 pipeline。最強自主 wow。
- **工時** ~1 天 · **風險中**(動到 orchestrator 核心)· **可棄守**(時間不夠先砍)。

### P0-4 ·【交付】送件影片 + 彩排(不可妥協)
- 主軸:「**啟動→離手→它按排程自行處理**」,用 audit 呈現觸發來源與決策軌跡。
- **工時** ~0.5 天。**沒影片 = 0 分。**

### P1 ·【強化自主】
- **健康監測 watchdog**:agent 自檢 Nemotron/Falcon/NemoClaw,將服務健康轉移記入監控首頁；復原仍由既有服務管理機制負責。
- **自主情勢簡報**:agent **按自己排程**(非人問)定時用 Nemotron 產「過去 1 小時:2 起,最嚴重 ch5…」推播。

### P2 ·(預設不做,列未來)
自然語言操作台(human-in-the-loop,**與主軸相左,降為選配**)、跨攝影機關聯、向量檢索、語音。

## 7. 各項驗收標準(Definition of Done)

- **P0-1 自主處置 + 自動報告**:確認 high/critical 事件時,**無人工核准**下,agent 依 NemoClaw 政策執行分級動作;`escalate` 產生可見二級告警;事件可形成 Markdown 報告與持久化紀錄;新增單元測試覆蓋處置/報告分支且全綠。
