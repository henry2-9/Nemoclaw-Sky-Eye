## 0. 北極星重校(這版的關鍵)
評審主軸是 **autonomous / no human in the loop**。上一版把「自然語言操作台(人去問它)」當主力是**方向錯誤**——那是 human-in-the-loop。本版一切圍繞:

> **「啟動後,人就走開。它自己看、自己想、自己調查、自己治理、自己決策、自己處置、自己報告、自己維生——並能**證明**全程 0 人工介入。」**

## 1. 戰略判斷
**已具備:** 真 NemoClaw 治理、防注入 + 5/5 攻擊矩陣、7 階段 flight recorder、世界 live CCTV、單台 GB10 級聯、SQLite、玻璃 dashboard、73 測試、supervisor 7×24 自跑。

**第一名差距(兩點):**
1. **自主性要「更深 + 可證明」**:目前閉環偏「偵測→通知」一槍;要升級成「偵測→**自主調查**→治理→**自主處置/報告**→**自我維生**」的完整閉環,且 dashboard 要**證明** 0 人工介入。
2. **要直觀**:評審 10 秒要看懂「這是一個自己在跑的 AI 保全官」,不是一張要讀的表。

## 2. 致勝定位(一句話 + 五支柱)
> **「NemoClaw Sentinel:單台 GB10 上 7×24 全自主的 AI 保全官。啟動即離手——Nemotron 看與想、不確定時自己深入調查、真 NemoClaw 治理每個動作、自主分級處置與產報告、還會自我監控維生,連攻擊它本身都自主擋下。全程可證 0 人工。」**

| 支柱 | 證據 | 狀態 |
|---|---|---|
| 🔁 **全自主閉環** | 偵測→調查→治理→處置→報告,無人 | ⛏ 深化 |

| 🩺 **自我維生** | 服務自檢、優雅降級、自動復原(watchdog) | ⛏ 強化+可視 |

## 3. 優先序(全以「推進自主性」為準)

### P0-1 ·【閉環】自主分級處置 + 自動事件報告
- **做什麼**:agent 不止通知。確認事件後**自主**執行分級動作(log / notify / **escalate 二級告警** / **自動產 `sentinel-violation-report` PDF 並送出**),全由 NemoClaw 政策決定、無人核准。
- **為何贏**:把 autonomous 從「會看會說」推到「**會做**」——閉環關鍵。
- **工時** ~0.5–1 天 · 低風險(串接既有 violation-report + policy actions)。

### P0-2 ·【可證明 + 直觀】自主運行監控首頁 + 「0 人工」證明
- **做什麼**:新首頁(B 重新定位為**自主運行觀測窗**,非互動控制台):
  - 大字證明:**「全自主運行 · 人工介入 0 次 · 已連續 Xh · 處理 N 起」**(由 audit 推算:無任何 human-triggered 動作)。
  - **live 攝影機格**(週期快照,不連續解碼)+ **威脅等級** banner。
  - **agent 思考即時流**:最新事件級聯逐步動畫(① Falcon→② Nemotron→③ NemoClaw→④ 政策→⑤ 處置),讓「它自己在跑」看得見。
- **為何贏**:10 秒看懂 + **證明**零人工(評審最在意的主軸,直接擺在臉上)。
- **工時** ~1 天 · 風險:抓幀效能(僅週期快取)· 複用:flight recorder、analyze 抓幀、glass UI。

### P0-3 ·【agentic 深度】自主調查(不確定就自己挖)
- **做什麼**:Nemotron 信心邊界 / 場景含糊時,agent **自主**再調查——多抓幾幀 / 換角度 / 追問 Nemotron——再決策。調查步驟記入 flight recorder。
- **為何贏**:這才是「agent」——自我導向多步推理,不是固定 pipeline。最強自主 wow。
- **工時** ~1 天 · **風險中**(動到 orchestrator 核心)· **可棄守**(時間不夠先砍)。

### P0-4 ·【交付】送件影片 + 彩排(不可妥協)
- 主軸:「**啟動→離手→它全自己處理**」,用 audit 證明 0 人工。
- **工時** ~0.5 天。**沒影片 = 0 分。**

### P1 ·【強化自主】
- **自我維生 watchdog**:agent 自檢 Nemotron/Falcon/NemoClaw,掛了優雅降級 + 自動重試/復原,事件記入監控首頁(「它讓自己活著」)。
- **自主情勢簡報**:agent **按自己排程**(非人問)定時用 Nemotron 產「過去 1 小時:2 起,最嚴重 ch5…」推播。

### P2 ·(預設不做,列未來)
自然語言操作台(human-in-the-loop,**與主軸相左,降為選配**)、跨攝影機關聯、向量檢索、語音。

## 5. 影片 demo 流程(證明「自主」,~3 分鐘)
1. **啟動即離手**:`systemctl status nemoclaw-sentinel` 7×24 自跑;強調無人觸發、無人確認。
2. **自主監控首頁**:「全自主 · 人工介入 0 次 · 連續 Xh」+ live 攝影機格 + 威脅等級。
3. **看它自己跑一起事件**:Falcon 候選 →(不確定)→ **agent 自主調查** → Nemotron 確認 → NemoClaw 治理 → **agent 自動 escalate + 自動產報告** → 全程無人。
4. **真 NemoClaw 治理**:governed_by=nemoclaw-openshell。
5. **自主防禦**:`demo_attack_scene.sh`——連治理模型被 OCR 騙都被 guardrail 自主否決;攻擊矩陣 5/5。
6. **自我維生**:示範一個服務中斷,agent 自己降級/復原(監控首頁顯示)。
7. **可證 0 人工**:audit 顯示全部動作皆 agent 觸發、exactly-once、flight recorder 全軌跡。
8. **收尾**:「啟動後零人工——自己看、想、查、守、處置、維生。」

## 7. 各項驗收標準(Definition of Done)

- **P0-1 自主處置 + 自動報告**:確認 high/critical 事件時,**無任何人工核准**下,agent 依 NemoClaw 政策執行分級動作;`escalate` 產生可見二級告警;自動產 `sentinel-violation-report` PDF 並(可選)送出;audit 每筆動作標記為 agent 觸發;新增單元測試覆蓋處置/報告分支且全綠。
- **P0-2 監控首頁 + 0 人工證明**:首頁顯示「**全自主 · 人工介入 0 次 · 連續 Xh · 處理 N 起**」(由 audit 推算,無任何 human-triggered 動作);含 live 攝影機格(週期快照、不連續解碼)、威脅等級、最新事件級聯動畫;**純觀測、無互動控制元件**。
- **P0-3 自主調查(可棄守)**:Nemotron 信心邊界/場景含糊時,agent 自動再抓幀或追問後才決策;flight recorder 顯示調查步驟;調查次數有上限(不無限);不破壞既有級聯測試。
- **P0-4 送件影片**:~3 分鐘,主線「啟動→離手→全自主處理」,以 audit 證明 0 人工;片中指令與實際啟動方式一致。
- **全程約束**:除 P0-3 外皆為加法、不動核心級聯/政策;每步保 **73+ 測試綠** 且 `demo_attack_scene.sh` 可過;新增 Nemotron 呼叫限併發、抓幀僅週期快取。
